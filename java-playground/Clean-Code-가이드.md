# 함수(메소드)
### 작게 만들어라
### 한 가지만 해라
### 함수 당 추상화 수준은 하나로
### 서술적인 이름을 사용하라
### 함수 인수
- 함수 인수는 적을수록 좋다
- 3개는 가능한 피하고, 4개 이상은 절대 사용하지 말것
- 인수가 2~3개 필요할 경우 클래스 사용을 검토
### Side Effect를 피하라
### 명령과 조회를 분리하라
- 함수는 수행하거나 조회하거나 둘 중 하나만 해야한다.
- 개체 상태를 변환하거나 아니면 개체 정보를 조회하거나 둘 중 하나다.
### 오류 코드보다 예외를 사용하라
- 함수는 '한 가지' 작업만 해야한다. 오류 처리도 '한 가지' 작업에 속한다.
- try/catch 블록은 코드 구조에 혼란을 일으킨다. try/catch 블록을 별도의 함수로 뽑아내는 편이 낫다.
```
public void delete(Page page) {
    try {
        deletePageAndAllReferences(page);
    } catch (Exception e) {
        logError(e);
    }
}
```
### 반복하지 마라
### 함수를 구현하는 방법
1. 서투른 코드를 작성한다.
2. 단위 테스트 코드를 작성한다.
3. 코드를 다듬는다. (이름 변경, 중복 제거, 메소드 간소화, 순서 변경, 클래스 분할)
